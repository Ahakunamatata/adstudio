## 1. 结论摘要

AD Studio Agent 的 UX 验收不应该按“聊天是否自然”验收，而应该按“用户是否能在可控成本下，把广告生产推进到可检查、可确认、可返工的画布产物”验收。

第一阶段应该把四条路径跑通到可验收状态：

| 路径 | 第一阶段目标 | 不做的事 |
| --- | --- | --- |
| 复刻广告 | 从产品包 + 竞品素材到复刻方案、脚本、锚点、分镜/视频生成确认 | 不承诺自动视觉 QA，不自动抄袭式复刻 |
| 从 0 创作广告 | 从一句广告目标到创意方向、脚本、锚点、分镜/视频生成确认 | 不让用户一次填完整大表 |
| 快速生成升级 Agent 项目 | 把 Ad Video / Ad Image 结果升级成可继续编排的 Agent 项目 | 不强制所有快速生成进入 Agent |
| 局部返工 | 对用户反馈做问题分类、影响节点识别、局部重跑确认 | 不默认整条链路重跑 |

核心体验判断：

- Agent 每次只追问阻塞下一步的问题。
- 所有高成本、不可逆、覆盖已确认结果的动作都必须出确认卡。
- 画布是事实源，聊天是控制台；重要产物必须落节点、状态、版本和引用关系。
- 失败状态要给出“发生了什么、影响哪些节点、用户现在能做什么”。

## 2. 通用 UX 验收对象

### 2.1 Agent 追问规则

Agent 的追问不能像表单补全，而要服务“最小可推进下一步”。

第一阶段追问优先级：

1. 没有产品包时，先问产品来源：产品 URL、从资产库选择、手动输入。
2. 没有输出边界时，只追问会影响生成成本或结果形态的信息：渠道、语言、比例、时长。
3. 复刻路径没有参考素材时，要求上传竞品图片/视频。
4. 复刻路径没有解析重点时，给多选按钮：Hook、脚本逻辑、剧情设计、构图、节奏、角色、CTA。
5. 用户要求直接生成但缺少高风险前置条件时，说明 blocker，并给最短补齐选项。

不应出现：

- 一次性要求用户填写完整广告 brief。
- 已有产品包、素材解析、画布节点时重复追问。
- 在用户只想快速生成时强行进入完整 Agent 流程。

### 2.2 确认卡片标准

确认卡片是 Agent UX 的核心控件，不是普通聊天消息。

第一阶段确认卡至少包含：

| 字段 | 说明 |
| --- | --- |
| 标题 | 用户能一眼知道要确认什么 |
| Agent 要做什么 | 动作摘要，例如“创建 5 个分镜图节点并发起生成” |
| 为什么现在做 | 对应前置条件已满足或用户刚确认了某产物 |
| 影响范围 | 受影响节点、下游节点、是否会标记 stale |
| 成本/时间 | 有 credits 动作时必须显示估算；未知时显示“待 provider 返回” |
| 可调整项 | 用户能改的参数，例如时长、比例、参考程度、CTA |
| 主操作 | 确认继续 |
| 次操作 | 修改、跳过、取消 |

确认卡必须用于：

- 创建批量生产节点。
- 发起图片/视频生成。
- 锁定产品、品牌、人物、场景、UI 锚点。
- 修改 locked 节点。
- 执行 repair plan。
- 导出或覆盖最终视频。

### 2.3 画布变化标准

画布每次变化都要能回答三个问题：

1. 这个节点是什么业务产物？
2. 它引用了哪些上游事实？
3. 它现在处于什么状态？

第一阶段节点状态建议：

| 状态 | UX 表现 | 用途 |
| --- | --- | --- |
| `draft` | 灰色/草稿标签 | Agent 生成但未确认的文本、方案、脚本 |
| `waiting_confirmation` | 黄色/待确认 | 等用户确认后才能推进 |
| `queued` | 排队中 | 生成任务已创建但未开始 |
| `running` | 进度/动效 | provider 生成中 |
| `succeeded` | 结果预览 | 生成或保存成功 |
| `failed` | 红色错误态 | 工具或 provider 失败 |
| `locked` | 锁图标 | 用户确认的锚点或最终结果 |
| `stale` | 橙色提示 | 上游变更导致下游需刷新 |

第一阶段画布分组：

- 产品核心锚点：产品包、Logo/Icon、产品图、App UI。
- 参考与策略：竞品素材、客观拆解、复刻方案、创意方向。
- 脚本与 Prompt：脚本、clip table、prompt pack。
- 一致性资产：人物、场景、品牌、字幕/voice。
- 视频生产：分镜图、分镜视频、最终视频。

### 2.4 失败卡片标准

失败不能只显示“生成失败”。失败卡必须把失败转成下一步选择。

失败卡至少包含：

| 字段 | 说明 |
| --- | --- |
| 失败类型 | 信息缺失、资产未完成、生成失败、品牌错误、人物漂移、场景污染、脚本方向错、字幕问题 |
| 发生位置 | 节点 ID / 节点标题 / 当前 playbook 阶段 |
| 影响范围 | 哪些下游节点 stale，哪些 locked 节点不受影响 |
| 可恢复性 | 可自动重试、需用户补信息、需人工确认、不可恢复 |
| 建议动作 | 补信息、重试当前节点、回退到脚本、回退到锚点、局部重跑 |
| 主操作 | 按推荐方式修复 |
| 次操作 | 查看详情、手动调整、取消 |

## 3. Playbook A：复刻广告路径

### 3.1 成功路径

```text
选择/创建产品包
→ 上传竞品图片/视频
→ 选择解析重点
→ 客观拆解
→ 复刻边界确认
→ 迁移复刻方案
→ 脚本确认
→ 锚点资产确认
→ 分镜 / prompt pack 确认
→ 生成视频
→ Review / Repair
```

### 3.2 用户输入

入口前用户需要提供：

- 产品包：产品 URL、资产库产品、或手动产品信息。
- 竞品素材：图片或视频。
- 解析重点：Hook、脚本逻辑、剧情设计、画面构图、节奏、角色、CTA。
- 复刻偏好：严格剧情、只保留结构、只参考视觉风格。
- 输出参数：渠道、语言、比例、时长。

可以后补的信息：

- 目标受众。
- 本地化语气。
- CTA 文案。
- 禁用表达，例如不能提具体折扣或医疗承诺。

### 3.3 Agent 追问

Agent 第一轮追问只问阻塞项：

- 如果没有产品包：`要复刻到哪个产品？可以粘贴产品 URL，或从资产库选择。`
- 如果没有竞品素材：`请上传要复刻的竞品图片或视频。`
- 如果没有解析重点：`你最想复刻这条广告的哪部分？`
- 如果用户要求“尽量一样”：`你希望严格复刻剧情，还是只保留结构和节奏？`
- 如果产品功能缺失：`竞品里出现了 X，但当前产品包没有对应功能。是否删除、替换为 Y，还是补充产品事实？`

### 3.4 确认卡片

复刻路径第一阶段至少需要 7 类确认卡：

| 卡片 | 触发时机 | 必须展示 |
| --- | --- | --- |
| 产品包确认卡 | 产品解析或选择后 | 名称、Logo/Icon、卖点、人群、禁用声明 |
| 素材使用确认卡 | 竞品上传后 | 素材预览、媒体类型、是否进入解析 |
| 解析重点卡 | 解析前 | 多选重点、参考程度 |
| 客观拆解确认卡 | 解析后 | Hook、剧情、镜头、CTA、不确定项 |
| 复刻方案卡 | 迁移方案生成后 | 保留项、替换项、风险、3 个方向 |
| 脚本确认卡 | 脚本生成后 | 镜头/口播/字幕/CTA，产品事实校验 |
| 生成执行确认卡 | prompt pack/workflow plan 生成后 | 节点数、模型、成本、时间、影响范围 |

### 3.5 画布变化

第一阶段画布应按以下节奏变化：

| 阶段 | 新增/更新节点 | 状态变化 |
| --- | --- | --- |
| 产品包确认 | `product_pack`、`product_asset` | 确认后 `locked` |
| 素材上传 | `competitor_asset` | 上传后 `succeeded` |
| 客观拆解 | `competitor_analysis` | 先 `draft`，用户确认后 `locked` |
| 复刻方案 | `clone_strategy` | 方向选择后 selected 版本 `locked` |
| 脚本 | `ad_script` | 修改中 `draft`，确认后 `locked` |
| 锚点资产 | `character_reference`、`scene_reference`、`product_asset`、`ui_anchor` | 生成中 `running`，确认后 `locked` |
| 分镜/prompt | `storyboard_frame`、`shot_prompt` | 分镜确认前 `waiting_confirmation` |
| 视频 | `shot_video`、`final_video` | 生成中 `running`，成功后 `succeeded` |

关键要求：

- 竞品拆解和迁移方案必须是两个不同产物。
- 复刻方案节点必须明确 `preserveElements` 与 `replaceElements`。
- 任何从竞品中迁移到当前产品的功能，都必须能追溯到产品包或用户确认。

### 3.6 失败状态

| 失败 | UX 表现 | 恢复动作 |
| --- | --- | --- |
| 竞品素材无法解析 | 素材节点 `failed`，错误卡说明媒体问题 | 重新上传、换图片关键帧、手动描述 |
| 解析结果缺用户重点 | 解析卡标记“未覆盖重点” | 追加重点后重跑分析 |
| 竞品功能与产品不匹配 | 复刻方案卡显示冲突项 | 删除该桥段、替换卖点、补充产品事实 |
| 用户不认可复刻方向 | `clone_strategy` 保持 draft | 重新生成 3 个方向或切换参考程度 |
| 锚点不一致 | 对应 anchor 节点 `failed` 或未 locked | 重新生成/上传锚点，不推进视频 |
| 视频生成失败 | 视频节点 `failed` | 重试当前节点或调整 prompt pack |
| 结果像竞品过度 | Review 卡提示版权/相似度风险 | 降低参考程度，重做视觉风格或剧情细节 |

### 3.7 第一阶段与后置

第一阶段必须实现：

- 产品包确认、竞品上传、解析重点选择。
- 客观拆解与迁移复刻分离展示。
- 复刻方案确认卡。
- 脚本确认卡。
- 生成前成本/影响范围确认。
- 画布节点状态、版本、locked/stale/failed 基础表现。
- 失败卡与最小恢复动作。

可以后置：

- 自动抽帧视觉 QA。
- 相似度/侵权风险自动评分。
- 多方向批量 A/B 变体生成。
- 节点级分支对比。
- 复刻策略模板市场。

## 4. Playbook B：从 0 创作广告路径

### 4.1 成功路径

```text
选择/创建产品包
→ 设置广告边界
→ Agent 生成创意方向 x3
→ 用户选择方向
→ 脚本确认
→ 锚点资产确认
→ 分镜 / prompt pack 确认
→ 生成视频
→ Review / Repair
```

### 4.2 用户输入

入口前最小输入：

- 产品包。
- 广告目标一句话。
- 渠道、语言、比例、时长中的至少一部分。

Agent 可从产品包推导但需要确认的信息：

- 核心卖点。
- 目标受众。
- 痛点。
- CTA。
- 本地化语气。

### 4.3 Agent 追问

追问策略：

- 如果广告目标太泛：`这条广告更想主打安装、试用、购买，还是品牌认知？`
- 如果渠道缺失：`主要投放 TikTok、Meta Reels，还是 YouTube Shorts？`
- 如果语言/国家缺失：`目标市场和语言是什么？`
- 如果产品包卖点过多：`这条 15s 广告优先打哪个卖点？`
- 如果用户说“你决定”：Agent 可以给默认建议，但必须在确认卡标记“系统推荐，可修改”。

### 4.4 确认卡片

从 0 创作路径第一阶段至少需要 6 类确认卡：

| 卡片 | 触发时机 | 必须展示 |
| --- | --- | --- |
| 产品包确认卡 | 产品导入/选择后 | 产品事实、可用资产、禁用声明 |
| 广告边界确认卡 | 进入工作台前或首轮 | 渠道、语言、比例、时长、目标 |
| 创意方向选择卡 | Agent 生成方向后 | 3 个方向、Hook、适用人群、风险 |
| 脚本确认卡 | 方向确认后 | 时间轴、口播/字幕、CTA |
| 锚点确认卡 | 脚本确认后 | 需要哪些人物、场景、产品/UI 锚点 |
| 生成执行确认卡 | prompt/workflow plan 后 | 节点数、成本、时间、可调整项 |

### 4.5 画布变化

| 阶段 | 新增/更新节点 | 状态变化 |
| --- | --- | --- |
| 产品包 | `product_pack`、`product_asset` | 确认后 `locked` |
| 广告边界 | `creative_brief` 或 `text` 节点 | 确认后 `locked` |
| 创意方向 | `creative_concept` x3 | 选中方向 `locked`，未选方向保留 draft |
| 脚本 | `ad_script` | 用户确认后 `locked` |
| 锚点 | `character_reference`、`scene_reference`、`ui_anchor` | 确认后 `locked` |
| 分镜 | `storyboard_frame`、`shot_prompt` | 确认后进入生成 |
| 视频 | `shot_video`、`final_video` | 成功后可设最终版本 |

### 4.6 失败状态

| 失败 | UX 表现 | 恢复动作 |
| --- | --- | --- |
| 产品包信息不足 | 产品包卡显示缺失字段 | 补 URL、上传资产、手动补卖点 |
| 创意方向过泛 | 方向卡标记“差异不足” | 重新生成方向，指定受众/痛点 |
| 用户不认可卖点 | 脚本方向错失败卡 | 回退到 creative plan |
| 脚本超时长 | 脚本卡显示时间超限 | 压缩镜头、减少卖点、改时长 |
| 锚点缺失 | 锚点卡显示 blocker | 上传或生成锚点 |
| 生成结果不符合脚本 | Review 卡关联 prompt pack | 回退 prompt pack 或分镜节点 |

### 4.7 第一阶段与后置

第一阶段必须实现：

- 一句话广告目标进入工作台。
- 广告边界确认。
- 3 个创意方向选择。
- 脚本确认、锚点确认、生成确认。
- 产品事实不能被模型擅自扩写。

可以后置：

- 平台规范自动校验。
- 多语言本地化质量评分。
- 自动生成多受众版本。
- 投放平台导出适配。

## 5. Playbook C：快速生成升级为 Agent 项目路径

### 5.1 成功路径

```text
Ad Video / Ad Image 快速生成
→ 生成结果成功
→ 用户点击“升级为 Agent 项目”
→ 选择升级目标
→ 补齐产品包/广告目标
→ 创建 Agent 项目和画布节点
→ Agent 给下一步建议
```

升级目标不应只有一个：

| 升级目标 | 适用场景 |
| --- | --- |
| 继续做变体 | 用户喜欢结果，想批量出不同 Hook/CTA |
| 作为锚点资产 | 把图片/视频用作人物、场景、产品或风格参考 |
| 扩展成完整广告 | 从单张图/单条视频继续做脚本、分镜、最终视频 |
| 复刻这个结果 | 把自己的快速生成结果当作参考结构进行迁移 |

### 5.2 用户输入

快速生成已有输入：

- prompt。
- 参考素材。
- 输出类型：image/video。
- 比例、模型、时长等参数。
- 生成结果 asset。

升级时需要补齐：

- 产品包：可选已有产品、导入 URL、手动创建。
- 升级目标。
- 是否保留原 prompt 作为事实，还是允许 Agent 重写。
- 结果在新项目中的角色：final draft、reference asset、anchor、concept。

### 5.3 Agent 追问

升级时 Agent 不应重新盘问完整 brief，只追问：

- `这个结果要作为最终广告草稿、参考风格，还是某个锚点资产？`
- `要绑定哪个产品包？`
- `是否允许我改写原 prompt 来生成更完整的分镜和脚本？`
- `要继续做视频、图片变体，还是进入完整 Agent 工作台？`

### 5.4 确认卡片

快速生成升级路径第一阶段至少需要 4 类确认卡：

| 卡片 | 触发时机 | 必须展示 |
| --- | --- | --- |
| 保存结果卡 | 快速生成成功后 | 结果预览、参数、保存到资产库 |
| 升级项目卡 | 用户点击升级后 | 新项目名称、绑定产品、升级目标 |
| 角色绑定卡 | 创建画布前 | 该结果作为 reference / anchor / draft / final 的哪一种 |
| 下一步建议卡 | 进入 Agent 后 | 继续变体、补脚本、生成锚点、生成视频 |

### 5.5 画布变化

| 阶段 | 新增/更新节点 | 状态变化 |
| --- | --- | --- |
| 快速生成成功 | `image` 或 `video` 资产 | `succeeded` |
| 保存资产 | 资产库记录 | 可被 Agent 引用 |
| 升级项目 | 新建 Agent session | 生命周期进入 `intake` 或 `ready` |
| 角色绑定 | `reference_asset` / `anchor` / `concept` 节点 | 根据用户确认可 `locked` |
| 下一步计划 | `creative_brief` 或 `repair_plan` | `draft`，等待确认 |

关键要求：

- 快速生成原参数必须保留为 provenance。
- 升级后不能丢失 prompt、参考素材、模型、比例、时长、assetUrl。
- 如果没有产品包，升级项目可以先创建，但进入生成前必须补齐。

### 5.6 失败状态

| 失败 | UX 表现 | 恢复动作 |
| --- | --- | --- |
| 快速生成失败 | 表单内失败卡 | 修改 prompt/素材/模型后重试 |
| 保存资产失败 | 结果仍可预览，保存卡失败 | 重试保存，不重复生成 |
| 升级缺产品包 | 升级卡 blocker | 选择产品包或先建空项目 |
| 角色绑定冲突 | 结果不适合作为某类锚点 | 改成 reference asset 或 concept |
| 原 prompt 太弱 | Agent 标记上下文不足 | 允许 Agent 补问或重写 brief |

### 5.7 第一阶段与后置

第一阶段必须实现：

- 快速生成成功结果可保存。
- 结果可升级为 Agent 项目。
- 升级时保留参数和 asset provenance。
- 用户可选择结果角色。
- Agent 进入项目后给出下一步确认卡。

可以后置：

- 一键从快速视频反推完整脚本。
- 自动识别结果最适合作为何种锚点。
- 快速生成历史批量升级。
- 多结果合并成一个 Agent 项目。

## 6. Playbook D：局部返工路径

### 6.1 成功路径

```text
用户指出问题
→ Agent 分类问题
→ 定位受影响节点
→ 生成 repair plan
→ 用户确认
→ 标记下游 stale
→ 局部重跑
→ 新旧版本对比
→ 用户确认是否替换主版本
```

### 6.2 用户输入

用户可能通过三种方式发起返工：

- 自然语言：`icon 不像我们的 App`、`这个人物不一致`、`字幕挡住 UI`。
- 点击节点操作：重跑、替换输入、锁定、分支。
- 在结果上标记问题：选中视频/图片节点后提交反馈。

第一阶段需要支持的反馈类型：

| 类型 | 示例 | 默认回退点 |
| --- | --- | --- |
| 品牌/UI 错误 | Logo 不准、App UI 错 | brand/ui anchor |
| 人物漂移 | 分镜人物不像同一个人 | character anchor |
| 场景污染 | 背景出现多余人物或错误场景 | scene anchor / prompt pack |
| 脚本方向错 | 卖点不对、CTA 不合适 | creative plan / script |
| 字幕问题 | 遮挡、语言错、太长 | subtitle / video assembly |
| 生成失败 | provider timeout、任务失败 | 当前 generation node |

### 6.3 Agent 追问

Agent 不应直接重跑，先判断是否能定位：

- 如果用户没指定节点：`这个问题出现在最终视频、某个分镜，还是脚本里？`
- 如果问题模糊：`是产品事实不对、画面不对，还是字幕/CTA 不对？`
- 如果会影响 locked 节点：`修复会修改已锁定的 X 锚点，是否允许？`
- 如果可局部修：`我可以只重跑 C3 和最终合成，不动产品包和脚本。是否继续？`

### 6.4 确认卡片

局部返工路径第一阶段至少需要 4 类确认卡：

| 卡片 | 触发时机 | 必须展示 |
| --- | --- | --- |
| 问题分类卡 | 用户反馈后 | 问题类型、置信度、可修改范围 |
| 影响范围卡 | 定位节点后 | 受影响节点、下游 stale、locked 节点保护 |
| 返工计划卡 | repair plan 生成后 | 回退点、重跑节点、成本/时间 |
| 版本替换卡 | 新结果生成后 | 新旧版本预览、是否设为主版本 |

### 6.5 画布变化

| 阶段 | 新增/更新节点 | 状态变化 |
| --- | --- | --- |
| 用户反馈 | 当前节点记录 feedback | 不立即改输出 |
| 问题分类 | `repair_plan` | `draft` |
| 确认返工 | 受影响节点 | `queued` / `running` |
| 上游变更 | 下游节点 | `stale` |
| 生成成功 | 目标节点新增 version | 新版本 `succeeded`，旧版本保留 |
| 替换主版本 | 目标节点 | 主版本切换，必要时 `locked` |

关键要求：

- 返工不能覆盖旧结果，必须追加版本。
- locked 节点默认受保护。
- 如果必须修改 locked 节点，确认卡要明确说明影响范围。
- 下游 stale 是 UX 必须可见的状态，不只是内部标记。

### 6.6 失败状态

| 失败 | UX 表现 | 恢复动作 |
| --- | --- | --- |
| 无法分类问题 | 问题分类卡置信度低 | 追问用户选择问题类型 |
| 找不到节点 | 反馈卡 blocker | 让用户点选节点或选择最终视频 |
| locked 节点冲突 | 影响范围卡警告 | 请求解锁确认或改用下游修复 |
| 局部重跑失败 | 目标节点 failed | 重试当前节点或扩大回退范围 |
| 修复后引入新问题 | 新版本不自动替换 | 保留旧主版本，继续 repair |
| 下游 stale 未刷新 | 画布提示未同步 | 生成刷新计划 |

### 6.7 第一阶段与后置

第一阶段必须实现：

- 用户可对节点/最终视频提交反馈。
- Agent 能给问题分类和影响范围。
- repair plan 必须经用户确认。
- 局部重跑追加版本，不覆盖旧版本。
- 下游 stale 可见。

可以后置：

- 自动视觉差异检测。
- 视频逐帧标注。
- 多版本并排对比高级交互。
- 自动选择最佳版本。
